Three workflows

The recurring moves of the job, done with rigor.

Not a sequence, a toolkit. Each runs standalone, powered by the same method core in prompts.py.

workflow / issue-tree

Issue Tree

Turns a business question into a MECE, hypothesis-driven tree. Each leaf a testable hypothesis with the analysis to run.

workflow / market-sizing

Market Sizing

Triangulates top-down and bottom-up, reconciles into a defensible range, and flags the assumptions it’s most sensitive to.

workflow / synthesize

Synthesize

Structures raw material by insight. A “so what” for every finding, plus the open questions the source doesn’t answer.

Real output

Every workflow returns a boardroom-ready exhibit.

Not a wall of text, but structured artifacts a consultant would actually put in a deck.

“AI drafts the first version. The consultant owns the judgment.”
Where the moat lives

The value isn’t the model call. It’s the method encoded in the prompts. StratKit enforces MECE structure, demands a falsifiable hypothesis per leaf, and refuses to invent facts. That’s the part that needs someone who knows the craft.

Provider-agnostic (Claude by default), UI-agnostic core, low temperature for defensible output.

# stratkit/prompts.py · the method in code
ISSUE_TREE_SYSTEM = """You are a senior strategy consultant.
Decompose into 2-4 MECE branches, no overlap, no gaps.
For every leaf: ONE falsifiable hypothesis + the ONE
analysis that would confirm or reject it."""

30-second setup

Use them right inside Claude.

Each skill is a folder with a SKILL.md (name + description). Claude loads it only when your prompt matches, so it costs nothing until it helps.

Claude apps · claude.ai
  1. Zip the skill folder (the zip must contain the folder, not just the file).
  2. Go to Settings → Capabilities → Skills → Upload and pick the zip.
  3. Ask your question in chat. Claude runs the skill automatically when it fits.
cd skills
zip -r issue-tree.zip issue-tree/
Claude Code
  1. Copy the skill folder into your skills directory.
  2. Claude Code discovers it on the next run. No restart needed.
cp -r skills/issue-tree \
  ~/.claude/skills/
